# Neon Ape v0.2.0

First public feature release.

## Highlights

- Interactive terminal startup with menu-driven actions
- Safe wrappers for `nmap`, `subfinder`, `dnsx`, `httpx`, `naabu`, and `nuclei`
- Chained recon workflow: `subfinder -> httpx -> naabu`
- Local SQLite-backed checklist, scan history, findings, and export/import flows
- Built-in uninstall and DB inspection commands
- History cleanup command for older chained workflow rows

## Commands to Try

```bash
neonape
neonape --workflow pd_chain --target example.com
neonape --tool nuclei --target https://example.com
neonape db scans
neonape db cleanup-history
```

## Notes

- Local-only by design. No web server or exposed listener.
- Targets are masked by default in landing and DB scan views unless explicitly revealed.
- This release focuses on authorized recon and scan workflows, not exploitation automation.
