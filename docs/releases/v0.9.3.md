# Neon Ape v0.9.3

Production polish release focused on automatic operations.

## Highlights

- New `neonape checklist --target <domain> --auto` command
- Adam now runs the MAGI checklist without `Run now?` prompts
- Managed recon bootstrap with `neonape setup tools --yes`
- Built-in passive recon module for WHOIS and DNS A-record collection
- Hardened Nmap XML parsing so empty or malformed XML no longer crashes MAGI

## Recommended Commands

```bash
neonape setup tools --yes
neonape adam --target stoic.ee --pdf
neonape checklist --target stoic.ee --auto
neonape autoresearch --target angel-eyes --auto --test-target stoic.ee --pdf
neonape status
```

## Notes

- Local-only by design
- Fernet-encrypted local data flows remain unchanged
- MAGI now prefers automation for tool-backed steps while preserving manual review steps where appropriate
