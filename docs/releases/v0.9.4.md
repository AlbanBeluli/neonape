# Neon Ape v0.9.4

Web-path enumeration release focused on a safer ffuf-first default.

## Highlights

- ffuf is now the default web-path enumeration engine
- Gobuster is preserved as an automatic legacy fallback for one version
- Adam and `pd_web_chain` now follow `subfinder -> httpx -> katana -> ffuf -> nuclei`
- Angel Eyes, review views, DB views, and Obsidian exports now accept ffuf path findings
- Managed tool setup now includes ffuf

## Recommended Commands

```bash
neonape setup tools --yes
neonape adam --target example.com --pdf
neonape adam --target example.com --use-gobuster
neonape --tool ffuf --target https://example.com
neonape --workflow pd_web_chain --target example.com
```

## Notes

- ffuf remains local-only and uses the same validated execution model as the rest of Neon Ape
- If ffuf cannot run, Neon Ape falls back to the preserved Gobuster path automatically
- Gobuster code is intentionally retained for one release cycle while ffuf becomes the primary path
