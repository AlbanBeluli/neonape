# Neon Ape v0.9.5

Reliability release focused on making ffuf the rock-solid default.

## Highlights

- ffuf now always uses a local wordlist at `~/.neonape/wordlists/common.txt`
- Local wordlist creation now works even offline through a small built-in fallback list
- ffuf defaults were tuned to `-t 40` and `-rate 80` for more stable runs
- `neonape setup tools --yes` now ensures the ffuf wordlist is present
- Fixed ffuf output handling so the default Adam web chain stays on ffuf instead of falling back

## Recommended Commands

```bash
neonape setup tools --yes
neonape adam --target example.com
neonape adam --target example.com --pdf
neonape --tool ffuf --target https://example.com
```

## Notes

- ffuf remains the default web-path engine
- Gobuster is still preserved as the legacy last-resort fallback for this release cycle
- The ffuf stage now logs the exact local wordlist path it is using
